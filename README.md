# Тестовое задание на должность Python Backend разработчика

## Запуск:
```sh
docker compose up -d --build
uv sync
alembic upgrade head
uv run -m src.seed
```

## Описание
Реализовано backend-приложение - система аутентификации и авторизации на основе RBAC.

В приложении используется пара токенов(access и refresh) для управления сессиями пользователей.

Всего есть 3 вида действий и 3 вида ресурсов. У каждой роли к каждому ресурсу имеется свой уровень доступа, определяемый полями can_read, can_write, can_delete.

В случае ошибки определения пользователя выдается 401, а в случае доступа к ресурсу, доступ к которому недоступен при данной роли выдается 403.

Data seeding закидывает вот таких 3 пользователей(почта / пароль)
- admin@example.com / admin123
- moderator@example.com / moderator123
- user@example.com / user123
